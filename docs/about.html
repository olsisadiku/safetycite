SafetyCite

An OSHA Compliance Copilot that cites the law — verifiably.

Small language models, fine-tuned with reinforcement learning, that answer workplace-safety questions and cite the correct OSHA standard (29 CFR) — where every citation is checked against the real federal regulation.

MinT · Tinker-compatible LoRA RL GRPO · verifiable reward Qwen3 · 0.6B → 4B FastAPI + React/Vite/Tailwind Live · no mocks

The idea

Most doc-chatbots can't tell you if they're right. OSHA can.

When someone asks about fall protection in construction, the correct answer cites a specific regulation — 29 CFR §1926.501 — and that citation is checkable ground truth, not a matter of opinion.

That makes OSHA the rare domain where reinforcement learning with a verifiable reward is the natural fit. SafetyCite trains the model with that exact signal: did it cite the right standard, in the right format, without inventing section numbers? Because the reward is objective, you get a hard, demoable result — citation accuracy goes up, hallucinated citations go down, measured on held-out questions.

How it works

From the real regulation to a verified answer

ASK ──▶ Router ──▶ Adapter (LoRA) ─┬─▶ plain-language answer + [29 CFR §xxxx.xx] └─▶ Corpus index ─▶ verify each citation exists + show its text TRAIN eCFR corpus ─▶ Q&A datasets ─▶ SFT ─▶ GRPO (citation reward) ─▶ adapter ─▶ eval ENGINES (all live, swappable) Local HF (MPS · CUDA · CPU) | MinT (remote GPUs)
1 · CORPUS

Real federal text

Pulls live 29 CFR sections (Construction 1926, General Industry 1910, Recordkeeping 1904) from the public eCFR API and indexes them. This index is the single source of truth for verifying citations.

2 · TRAIN

SFT, then GRPO

Supervised fine-tuning teaches the answer format; GRPO then samples several answers per question, scores each with the verifiable reward, and reinforces the better-cited ones.

3 · SERVE

Verified, live

A FastAPI server runs the real model, parses citations from each answer, and checks them against the corpus so the UI can show the actual regulatory text — and flag anything invented.

The verifiable reward

One deterministic function — no judge model

reward = citation F1  +  format  −  hallucination penalty

Citation F1

Set-overlap (F1, not recall) between cited and correct sections — so over-citing is penalised, not rewarded.

Format

Rewards a real plain-language answer plus a well-formed [29 CFR §…] citation the UI can parse.

Hallucination penalty

Every cited section must exist in the corpus. Inventing a section number is punished — this is what makes it trustworthy.

The same function is both the RL training signal and the evaluation metric — fully reproducible, with no second LLM grading the first.

Mixture of compliance experts

One adapter per OSHA domain

Instead of one model spread thin, SafetyCite trains a separate lightweight LoRA adapter per regulatory domain and routes each question to the right one — hot-swappable on a single shared base model.

Does training actually help?

Measured base → fine-tuned (held-out test sets)

DomainCitation F1FormatHallucinationReward
Construction (SFT→RL)0.00 → 0.170.45 → 1.000.70 → 0.10−0.26 → +0.32

The base Qwen3-0.6B gets 0% citation accuracy and invents sections 70–100% of the time. After training: perfect format, near-zero hallucination, and a clean climb in reward. Exact-citation accuracy is modest at 0.6B — it scales with model size (e.g. Qwen3-4B-Instruct on Colab/MinT), more RL steps, and more data.

The interface

Built to be inspected, not trusted blindly

  • Verifiable citations — every citation is a chip you can click to read the actual CFR text; a ✓/✗ badge shows whether the section is real.
  • Base vs fine-tuned — a side-by-side toggle that runs the un-trained model too, so the improvement is visible per question.
  • Auto-routing — the question is routed to the right domain adapter, and the UI shows which one answered and why.
  • Evaluation dashboard — base-vs-adapter metrics per domain (citation F1, format, hallucination) so the numbers above are live, not slideware.

Run it

Three ways, all running a real model

Primary path — Colab (free GPU, public URL). Open notebooks/colab_app.ipynb, pick a T4 GPU, run all cells: it clones, installs, fetches the corpus, trains the adapters on Qwen3-4B-Instruct-2507, builds the UI, serves the real model, and prints a public cloudflared link.
# Local (Apple Silicon / MPS)
uv venv && uv pip install -e ".[local]"
uv run safetycite fetch          # real OSHA text from eCFR
uv run safetycite build          # build Q&A datasets
uv run safetycite sft construction   # real LoRA fine-tuning
uv run safetycite rl  construction   # GRPO with the verifiable reward
uv run safetycite eval construction  # base vs adapter
uv run safetycite serve          # API + UI on :8000

# Remote MinT GPUs
export SAFETYCITE_BACKEND=mint TINKER_API_KEY=sk-mint-...
safetycite sft construction --backend mint

Under the hood

Training

MinT (Tinker-compatible) for remote LoRA SFT/GRPO, or Hugging Face transformers + peft locally. A custom GRPO loop scores groups of samples with the verifiable reward and back-props per sample to stay memory-light.

Serving & UI

FastAPI serves the real model and the corpus; a Bun + React + Vite + Tailwind front-end renders verified citations, the base/fine-tuned comparison, and the eval dashboard.

Data

Corpus and datasets are generated from the live eCFR API; every gold citation is validated against the corpus before it's used for training or evaluation.

Quality

The reward logic and corpus index are covered by unit tests; the whole pipeline is reproducible from a single CLI.
